# 迭代记录模板

每次代码更新、规则调整、架构修改、bug 修复、测试新增，都必须在D:\ocrwiki版本\single_main_nanozyme\docs\iteration_logs该文件夹中新增或更新一份 Markdown 记录。

## 文件命名规范

```
YYYY-MM-DD_short-title.md
```

## 记录格式

```md
# 更新标题

## 更新时间
YYYY-MM-DD HH:mm

## 更新类型
- 架构调整 / 功能开发 / Bug 修复 / 测试 / 文档 / 重构 / 规则更新

## 背景
这次为什么要改。

## 改动内容
- 改了什么文件
- 做了什么逻辑变化
- 新增了什么规则或能力

## 未改动内容
说明哪些相关部分没有动，避免误解。

## 验证方式
- 运行了什么命令
- 通过了什么测试
- 如果未验证，说明原因

## 风险与后续
- 可能的问题
- 下一步建议
```
