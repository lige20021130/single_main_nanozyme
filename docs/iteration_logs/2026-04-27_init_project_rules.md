### 建立的规则

**工程写作规则（三条核心规则）：**

1. **第一性原理** - 所有修改必须围绕项目目标，不引入旧系统的复杂逻辑
2. **先读代码，再理解，再实施** - 修改前必须阅读文件、理解调用链、找到最小修改点
3. **只改相关代码** - 禁止无关重构、顺手改风格、大范围格式化

**迭代记录规则：**

- 每次代码更新、规则调整、架构修改、bug 修复、测试新增，都必须新增或更新 Markdown 记录
- 文件命名：`YYYY-MM-DD_short-title.md`
- 记录必须包含：更新时间、更新类型、背景、改动内容、未改动内容、验证方式、风险与后续
